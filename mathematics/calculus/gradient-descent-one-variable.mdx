## Introduction

Gradient descent is a fundamental optimization algorithm used extensively in machine learning. This guide starts with simple examples of functions with one variable to build intuition about how the method works.

## Setup

```python
import numpy as np
import matplotlib.pyplot as plt

%matplotlib widget
```

## Function with One Global Minimum

Consider the function $f(x) = e^x - \log(x)$ (defined for $x > 0$), which has only one **global minimum** point.

### The Gradient Descent Algorithm

Starting from an initial point $x_0$, calculate the derivative (gradient) and step to the next point:

$$
x_1 = x_0 - \alpha \frac{df}{dx}(x_0)
$$

where:
- $\alpha > 0$ is the **learning rate**
- $\frac{df}{dx}(x_0)$ is the **gradient** at $x_0$
- The process repeats iteratively for $n$ iterations

<Note>
Subtracting the gradient moves "down the hill" toward the minimum. The gradient defines the direction, and $\alpha$ scales the step size.
</Note>

### Implementation

Define the function and its derivative:

```python
def f_example_1(x):
    return np.exp(x) - np.log(x)

def dfdx_example_1(x):
    return np.exp(x) - 1/x
```

Visualize the function:

```python
def plot_f(x_range, y_range, f, ox_position):
    x = np.linspace(*x_range, 100)
    fig, ax = plt.subplots(1, 1, figsize=(8, 4))
    
    ax.set_ylim(*y_range)
    ax.set_xlim(*x_range)
    ax.set_ylabel('$f\,(x)$')
    ax.set_xlabel('$x$')
    ax.spines['left'].set_position('zero')
    ax.spines['bottom'].set_position(('data', ox_position))
    ax.spines['right'].set_color('none')
    ax.spines['top'].set_color('none')
    
    ax.plot(x, f(x), 'k')
    return fig, ax

plot_f([0.001, 2.5], [-0.3, 13], f_example_1, 0.0)
plt.show()
```

### Gradient Descent Implementation

```python
def gradient_descent(dfdx, x, learning_rate=0.1, num_iterations=100):
    for iteration in range(num_iterations):
        x = x - learning_rate * dfdx(x)
    return x
```

### Running the Algorithm

```python
num_iterations = 25
learning_rate = 0.1
x_initial = 1.6

result = gradient_descent(dfdx_example_1, x_initial, learning_rate, num_iterations)
print(f"Gradient descent result: x_min = {result}")
```

## Understanding Parameters

### Learning Rate ($\alpha$)

The learning rate controls step size:

<Tabs>
  <Tab title="Too Small">
    ```python
    learning_rate = 0.04
    num_iterations = 25
    # Converges slowly, may not reach minimum
    ```
    **Result**: Slow convergence, needs more iterations
  </Tab>
  
  <Tab title="Good">
    ```python
    learning_rate = 0.1
    num_iterations = 25
    # Converges efficiently
    ```
    **Result**: Reaches minimum in ~21 iterations
  </Tab>
  
  <Tab title="Large">
    ```python
    learning_rate = 0.3
    num_iterations = 25
    # Faster but potentially unstable
    ```
    **Result**: Converges faster (~15 iterations)
  </Tab>
  
  <Tab title="Too Large">
    ```python
    learning_rate = 0.5
    num_iterations = 25
    # Overshoots and diverges
    ```
    **Result**: Does not converge!
  </Tab>
</Tabs>

### Initial Point ($x_0$)

The starting position affects convergence:

```python
# Different starting points
x_initial_1 = 1.6   # Works well
x_initial_2 = 0.05  # Works but steep gradient
x_initial_3 = 0.03  # Risk of overshooting
x_initial_4 = 0.02  # Diverges
```

<Warning>
When $x_0$ is very small, the function is steeper (larger gradient), leading to larger steps that may overshoot the minimum.
</Warning>

### Number of Iterations

```python
# Insufficient iterations
num_iterations = 10  # May not converge

# Adequate iterations
num_iterations = 25  # Good balance

# Excessive iterations
num_iterations = 75  # Wastes computation if already converged
```

<Tip>
Monitor the cost function to determine when to stop. If it stops decreasing significantly, you can terminate early.
</Tip>

## Function with Multiple Minima

Consider a more complex function with both local and global minima.

### The Challenge

With multiple minima, gradient descent may converge to:
- **Global minimum**: The best solution
- **Local minimum**: A suboptimal solution

The outcome depends on the initial point!

### Example

```python
# Function with multiple minima (predefined)
from w2_tools import f_example_2, dfdx_example_2

# Visualize
plot_f([0.001, 2], [-6.3, 5], f_example_2, -6)
plt.show()
```

### Different Starting Points

```python
learning_rate = 0.005
num_iterations = 35

# Starting from x = 1.3 (finds global minimum)
x_min_global = gradient_descent(dfdx_example_2, x=1.3, 
                                 learning_rate=learning_rate, 
                                 num_iterations=num_iterations)
print(f"Global minimum: x_min = {x_min_global}")

# Starting from x = 0.25 (gets stuck in local minimum)
x_min_local = gradient_descent(dfdx_example_2, x=0.25,
                               learning_rate=learning_rate,
                               num_iterations=num_iterations)
print(f"Local minimum: x_min = {x_min_local}")
```

## Key Insights

<AccordionGroup>
  <Accordion title="Learning Rate Trade-offs">
    - **Small**: Safe but slow, requires more iterations
    - **Medium**: Balanced convergence speed
    - **Large**: Fast but risks overshooting and divergence
    - Optimal value depends on the problem
  </Accordion>
  
  <Accordion title="Initial Point Importance">
    - Determines which minimum you find (for multiple minima)
    - Affects convergence speed
    - Steep regions require careful learning rate
    - Random initialization is common in ML
  </Accordion>
  
  <Accordion title="Convergence Criteria">
    Stop when:
    - Gradient is close to zero
    - Function value stops decreasing
    - Maximum iterations reached
    - Validation error increases (in ML)
  </Accordion>
</AccordionGroup>

## Visualization Example

```python
def visualize_gradient_descent(f, dfdx, x_initial, learning_rate, num_iterations):
    x_history = [x_initial]
    x = x_initial
    
    for i in range(num_iterations):
        x = x - learning_rate * dfdx(x)
        x_history.append(x)
    
    # Plot function
    x_plot = np.linspace(0.001, 2.5, 1000)
    plt.figure(figsize=(12, 5))
    
    plt.subplot(1, 2, 1)
    plt.plot(x_plot, f(x_plot), 'b-', linewidth=2, label='f(x)')
    plt.plot(x_history, [f(x) for x in x_history], 'ro-', 
             markersize=4, label='GD path')
    plt.xlabel('x')
    plt.ylabel('f(x)')
    plt.title('Gradient Descent Path')
    plt.legend()
    plt.grid(True, alpha=0.3)
    
    plt.subplot(1, 2, 2)
    plt.plot([f(x) for x in x_history], 'g-o', markersize=4)
    plt.xlabel('Iteration')
    plt.ylabel('f(x)')
    plt.title('Cost Function vs Iteration')
    plt.grid(True, alpha=0.3)
    
    plt.tight_layout()
    plt.show()

visualize_gradient_descent(f_example_1, dfdx_example_1, 
                           x_initial=1.6, learning_rate=0.1, num_iterations=25)
```

## Common Issues and Solutions

| Issue | Cause | Solution |
|-------|-------|----------|
| Divergence | Learning rate too large | Reduce $\alpha$ |
| Slow convergence | Learning rate too small | Increase $\alpha$ or add more iterations |
| Stuck in local minimum | Poor initialization | Try multiple random starts |
| Oscillation | Learning rate too large | Reduce $\alpha$ or use momentum |
| Premature stopping | Too few iterations | Increase $n$ or use adaptive stopping |

## Summary

Gradient descent is a powerful but sensitive optimization method:

- **Strengths**: Simple, widely applicable, foundation for ML
- **Weaknesses**: Requires tuning, can get stuck in local minima
- **Key parameters**: Learning rate, initial point, number of iterations
- **Best practices**: Monitor convergence, try multiple initializations, use adaptive methods for complex problems

Understanding gradient descent in one variable provides essential intuition for more complex optimization in machine learning.
