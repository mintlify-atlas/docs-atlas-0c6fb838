## Introduction

The cost function gives you a way to measure how well a specific set of parameters fits the training data. It allows you to evaluate and improve your parameter choices.

## Cost Function for Linear Regression

### The Squared Error Cost Function

For linear regression, we use the **squared error cost function**:

```math
J(w, b) = \frac{1}{2m} \sum_{i=1}^{m} (f_{w,b}(x^{(i)}) - y^{(i)})^2
```

Where:
- **m**: Number of training examples
- **f(x^(i))**: Prediction for the i-th example
- **y^(i)**: Actual value for the i-th example


### Understanding the Formula

Let's break down how this cost function works:

<Steps>
  <Step title="Compute the error">
    For each training example, calculate the difference between prediction and actual value:
    ```math
    \text{error} = \hat{y}^{(i)} - y^{(i)}
    ```
  </Step>
  
  <Step title="Square the error">
    Square this difference to penalize large errors more:
    ```math
    \text{error}^2 = (\hat{y}^{(i)} - y^{(i)})^2
    ```
  </Step>
  
  <Step title="Sum over all examples">
    Add up the squared errors for all training examples:
    ```math
    \sum_{i=1}^{m} (f_{w,b}(x^{(i)}) - y^{(i)})^2
    ```
  </Step>
  
  <Step title="Average the result">
    Divide by 2m to get the average (the 2 is for mathematical convenience):
    ```math
    J(w,b) = \frac{1}{2m} \sum_{i=1}^{m} (f_{w,b}(x^{(i)}) - y^{(i)})^2
    ```
  </Step>
</Steps>


<Note>
  The division by 2m (instead of just m) is a convention that makes the math cleaner when we compute derivatives for gradient descent.
</Note>

### Why Squared Error?

The squared error cost function:
- Penalizes large errors more than small errors
- Is always non-negative
- Is differentiable (smooth)
- Creates a convex optimization surface for linear regression


For linear regression, this creates a nice bowl-shaped (convex) surface, making it easy for gradient descent to find the global minimum.

## Cost Function for Logistic Regression

### Why Not Squared Error?

For logistic regression, the squared error cost function doesn't work well. If we use it, we get a **non-convex** cost function with many local minima:


<Warning>
  With a non-convex cost function, gradient descent can get stuck in local minima and fail to find the global minimum.
</Warning>

### The Logistic Loss Function

For logistic regression, we use a different cost function:


The loss for a single training example is:

```math
L(f(x), y) = \begin{cases}
-\log(f(x)) & \text{if } y = 1 \\
-\log(1 - f(x)) & \text{if } y = 0
\end{cases}
```

The overall cost function is:

```math
J(w, b) = \frac{1}{m} \sum_{i=1}^{m} L(f(x^{(i)}), y^{(i)})
```

### Intuition: When y = 1

When the true label is y = 1, the loss is -log(f(x)):


<Accordion title="Understanding the curve">
  - If f(x) = 1 (perfect prediction): loss ≈ 0
  - If f(x) = 0.5 (uncertain): loss is moderate
  - If f(x) → 0 (wrong prediction): loss → ∞
  
  This heavily penalizes confident wrong predictions!
</Accordion>

Zooming in on the relevant range (0 to 1):


Example: If the model predicts f(x) = 0.1 but y = 1:


The loss is very high, encouraging the model to predict values closer to 1.

### Intuition: When y = 0

When the true label is y = 0, the loss is -log(1 - f(x)):


<Accordion title="Understanding the curve">
  - If f(x) = 0 (perfect prediction): loss ≈ 0
  - If f(x) = 0.5 (uncertain): loss is moderate
  - If f(x) → 1 (wrong prediction): loss → ∞
</Accordion>

Example: If the model predicts f(x) = 0.999 (99.9% chance of malignant) but y = 0 (actually benign):


The loss is extremely high, heavily penalizing this confident wrong prediction.

### Simplified Cost Function

We can combine both cases into a single equation:

```math
L(f(x), y) = -y \log(f(x)) - (1-y) \log(1 - f(x))
```

<Note>
  When y = 1, the second term vanishes. When y = 0, the first term vanishes. This clever formulation combines both cases elegantly.
</Note>

The complete cost function is:

```math
J(w, b) = -\frac{1}{m} \sum_{i=1}^{m} [y^{(i)} \log(f(x^{(i)})) + (1-y^{(i)}) \log(1 - f(x^{(i)}))]
```

### Convexity

This cost function is **convex** for logistic regression:


This means gradient descent will reliably find the global minimum!

## Comparing Cost Functions

| Aspect | Linear Regression | Logistic Regression |
|--------|------------------|--------------------|
| **Cost Function** | Squared Error | Logistic Loss |
| **Formula** | (f(x) - y)² | -y log(f(x)) - (1-y) log(1-f(x)) |
| **Shape** | Convex | Convex |
| **Output Range** | [0, ∞) | [0, ∞) |
| **Penalizes** | Large errors | Confident wrong predictions |

## Implementing Cost Functions

### Linear Regression Cost (Python)

```python
import numpy as np

def compute_cost_linear(X, y, w, b):
    """
    Compute cost for linear regression
    """
    m = len(y)
    predictions = np.dot(X, w) + b
    cost = (1 / (2 * m)) * np.sum((predictions - y) ** 2)
    return cost
```

### Logistic Regression Cost (Python)

```python
def compute_cost_logistic(X, y, w, b):
    """
    Compute cost for logistic regression
    """
    m = len(y)
    z = np.dot(X, w) + b
    f = 1 / (1 + np.exp(-z))  # sigmoid
    
    cost = -(1/m) * np.sum(
        y * np.log(f) + (1 - y) * np.log(1 - f)
    )
    return cost
```

## Summary

Cost functions:
- Measure how well your parameters fit the training data
- Guide the learning algorithm toward better parameters
- Must be convex for gradient descent to work reliably

**For Linear Regression:**
- Use squared error cost function
- Creates a convex bowl-shaped surface
- Penalizes large errors

**For Logistic Regression:**
- Use logistic loss (cross-entropy)
- Creates a convex surface
- Heavily penalizes confident wrong predictions
