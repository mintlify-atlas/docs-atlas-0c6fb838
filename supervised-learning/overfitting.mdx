## Introduction

You've learned about linear regression and logistic regression. They work well for many tasks, but sometimes algorithms can run into problems called **overfitting** and **underfitting**, which cause poor performance.

In this section, we'll explore what these problems are and how to address them, particularly using a technique called **regularization**.

### The Goldilocks Principle


Like the story of Goldilocks and the Three Bears:
- Too few features → underfitting (high bias)
- Too many features → overfitting (high variance)
- Just right → good generalization

## Addressing Overfitting

Here are three main approaches to address overfitting:

### Option 1: Collect More Training Data


With more training examples, even a high-order polynomial will fit a smoother function:


<Note>
  Getting more data is the number one tool against overfitting. When available, it can work really well.
</Note>

However, more data isn't always available or practical to obtain.

### Option 2: Feature Selection

Reduce the number of features you're using.


Instead of using all 100 features, select only the most useful ones:


<Accordion title="Feature Selection Approaches">
  - Use your intuition to pick the most relevant features
  - Use algorithms that automatically select features (covered in Advanced Learning)
  - Domain expertise to identify important variables
</Accordion>

<Warning>
  One disadvantage of feature selection: you're throwing away information. If all features are useful for prediction, you don't want to discard some.
</Warning>

### Option 3: Regularization

Regularization is a technique to reduce overfitting without eliminating features.


<Note>
  **Regularization** encourages the learning algorithm to shrink parameter values without necessarily setting them to exactly 0. This keeps all features but prevents them from having an overly large effect.
</Note>

Even with high-order polynomials, if you keep parameter values small, you get a curve that fits better:


#### How Regularization Works

- Setting a parameter to 0 is equivalent to eliminating a feature
- Regularization gently reduces parameter impacts without eliminating features
- You can keep all features while preventing overfitting
- By convention, we regularize w₁, w₂, ..., wₙ but typically not b

#### Regularization in Practice

Modify your cost function to penalize large parameter values:

```math
J(w, b) = \frac{1}{2m} \sum_{i=1}^{m} (f_{w,b}(x^{(i)}) - y^{(i)})^2 + \frac{\lambda}{2m} \sum_{j=1}^{n} w_j^2
```

Where λ (lambda) is the **regularization parameter** that controls how much to penalize large weights.

## Key Takeaways

**Underfitting (High Bias):**
- Model is too simple
- Poor performance on training data
- Poor performance on new data
- Solution: Use more features or more complex model

**Overfitting (High Variance):**
- Model is too complex
- Excellent performance on training data
- Poor performance on new data
- Solution: More data, fewer features, or regularization

**Good Fit:**
- Balanced complexity
- Good performance on training data
- Good performance on new data
- Goal: Achieve this balance

## Regularization Benefits

<Note>
  Regularization is used extensively in practice, including for training neural networks. It's one of the most important techniques for building models that generalize well.
</Note>

Regularization:
- Keeps all features
- Prevents overfitting
- Works with neural networks and other algorithms
- Is easy to implement
- Requires tuning the regularization parameter λ

## Next Steps

To build better models:

1. **Understand your data**: Is your model underfitting or overfitting?
2. **Choose appropriate complexity**: Balance model complexity with available data
3. **Use regularization**: Especially when you have many features
4. **Validate properly**: Use separate validation data to check generalization
5. **Iterate**: Try different approaches and compare results
